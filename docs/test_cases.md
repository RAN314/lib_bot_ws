## 🎬 端到端测试（E2E Test Cases）

### TC_E2E_01: 完整找书流程

**前置条件：**
- Gazebo仿真环境已启动
- 图书馆中有50本测试图书
- 机器人位于Home位置
- 所有节点正常运行

**测试步骤：**
1. 用户通过UI输入book_id: `978-3-16-148410-0`
2. 系统查询数据库，获取图书位置（A3书架，2层，5本）
3. 机器人规划路径到A3书架（前方0.8米）
4. 机器人开始移动，沿路径导航
5. 机器人到达目标位置（误差<0.5米）
6. 四向RFID扫描器激活，扫描周围图书
7. 检测到目标图书RFID标签（信号强度>0.7）
8. UI显示找到图书的位置信息
9. 数据库更新图书状态为"found"
10. 任务完成，机器人返回Home

**预期结果：**
- 导航成功率：100%
- 检测成功率：>85%
- 总耗时：<90秒
- 到达精度：<0.5米误差

**验收标准：**
```
✓ 机器人成功导航到A3书架
✓ RFID检测到目标图书
✓ UI正确显示图书信息
✓ 数据库状态已更新
✓ 任务状态标记为completed
```

- 缺失检测：1本（准确识别）

**验收标准：**
```
✓ 报告中显示错架图书数量和位置
✓ 报告中显示缺失图书ID

```

### TC_E2E_05: 错误恢复流程

**前置条件：**
- 机器人导航中失去定位
- AMCL定位误差>2米

**测试步骤：**
1. 机器人触发定位丢失状态
2. Nav2自动执行恢复行为1：原地旋转360°
3. 如果仍无法定位，执行恢复行为2：清除代价地图
4. 如果仍无法定位，执行恢复行为3：后退0.5米重试
5. 反馈到UI："定位丢失，正在恢复..."
6. 成功恢复定位，继续任务
7. 恢复失败，任务标记为FAILED

**预期结果：**
- 前两种恢复行为成功率>70%
- 总恢复时间<30秒
- 不可恢复时优雅失败

**验收标准：**
```
✓ 尝试所有三种恢复行为
✓ 恢复成功继续执行原任务
✓ 恢复失败标记FAILED状态
✓ UI显示适当提示信息
```

## 📝 备注

### 测试工具
- **单元测试：** `pytest` + `unittest`
- **集成测试：** 自研测试框架（基于ROS2 launch）
- **性能测试：** `ros2 topic hz` + `rqt_plot`
- **覆盖率：** `coverage.py` + `lcov`

### 持续集成
建议在GitHub/GitLab配置CI/CD：
```yaml
# .github/workflows/test.yml
on: [push, pull_request]
jobs:
  test:
    - 自动运行单元测试
    - 生成覆盖率报告
    - 静态代码检查（flake8）
```
