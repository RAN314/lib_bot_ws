## 📋 UI设计决策汇总

## 📱 左侧面板 - 控制区

### 区域1：任务控制（占左侧60%）

**按钮功能：**

1. **找书按钮** 
   - 文本："🎯 找书"
   - 快捷键：Ctrl+N
   - 行为：
     - 弹出找书对话框
     - 输入book_id或书名
     - 确认后发送Action Goal
     - 自动更新主界面

2. **库存盘点按钮**
   - 文本："📚 盘点"
   - 快捷键：Ctrl+I
   - 行为：
     - 弹出区域选择
     - 选择A/B/C/D区或全部
     - 确认后开始盘点
     - 主界面切换为盘点进度

3. **暂停/继续按钮**
   - 文本："⏸️ 暂停" / "▶️ 继续"
   - 快捷键：Space
   - 行为：
     - 切换当前任务状态
     - 暂停：机器人停止，保留状态
     - 继续：恢复执行

4. **取消按钮**
   - 文本："⏹️ 取消"
   - 快捷键：Esc
   - 行为：
     - 弹出确认对话框
     - 确认后优雅取消任务
     - 清空当前任务显示

**状态显示：**
```
当前任务: #42
任务状态: 🟢 运行中 | 🟡 暂停 | 🔴 取消中
```

## 🚀 Qt与ROS2集成（高层设计）

### 架构模式
```
Qt主线程（UI）
   ↕︎ Signal/Slot
   
Ros2Manager（封装类）
   ↕︎ ROS2 API调用
   
ROS2 Nodes（独立进程）
   - bt_manager_node（BT决策）
   - nav2_bt_navigator（导航）
   - rfid_processor_nodes（RFID处理）
   - db_manager_node（数据库）
   
通信方式:
   - Action: /find_book, /perform_inventory
   - Service: /db/query_book_info
   - Topic: /nav2/feedback, /rfid/detection_status
```

### 线程安全设计
```yaml
共享数据:
  - 当前任务状态
  - 任务队列
  - 日志缓冲区

保护措施:
  - std::mutex (C++11) 或 QMutex (Qt)
  - 读取: lock() → 读 → unlock()
  - 写入: lock() → 写 → unlock()
  - 避免死锁: 统一锁的顺序
```

## 📈 性能优化

### 刷新频率优化
```yaml
高频率更新 (10Hz):
  - 导航进度
  - 机器人位置
  - 任务状态

中频率更新 (5Hz):
  - RFID信号强度
  - 扫描结果

低频率更新 (1Hz):
  - 统计信息
  - 日志消息 (事件触发)
  - 电池状态
```

### 内存管理
```yaml
日志缓冲区:
  - 队列: std::queue<string>
  - 最大容量: 100条
  - 超出后: pop_front() 删除最旧

任务历史:
  - 显示: 最近100个任务
  - 存储: SQLite数据库
  - 查询: 按需加载，不全部载入内存
```
