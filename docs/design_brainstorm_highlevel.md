## 🎯 项目范围

### 核心目标
构建图书馆机器人的ROS2仿真系统，验证以下核心功能：
- **自主导航**：在书架间安全移动
- **图书检索**：找到并定位目标图书
- **库存管理**：执行自动化盘点

### 非目标（明确排除）
- ❌ 物理抓取动作（服务型机器人）
- ❌ 多层建筑导航（仅限单层）
- ❌ Sim2Real迁移（仅限仿真）
- ❌ 多机器人协作（单机器人场景）

## 🏗️ 系统架构

### 技术选型

| 层面 | 选择 | 决策理由 |
|------|------|----------|
| **仿真平台** | Gazebo Classic | ROS2集成成熟、资源丰富 |
| **机器人平台** | TurtleBot3 Waffle Pi | 计算能力强、可扩展性好 |
| **数据存储** | SQLite | 轻量级、支持事务、无需服务器 |
| **导航框架** | Nav2 (ROS2) | 官方推荐、社区支持 |
| **通信方式** | Action + Service | 适合长时间任务和请求-响应 |

### 架构原则

1. **模块化设计**：感知、导航、任务、数据解耦
2. **事件驱动**：通过ROS 2 Topic实现松耦合
3. **实时数据库**：图书位置动态更新
4. **人机协作**：机器人定位 + 人工取书

## 📦 核心组件

### 1. 四向RFID感知系统

**设计决策：使用四向扫描（前/后/左/右）**

**Why Four-Direction？**
- 360°覆盖 vs 方向明确：四向扫描在两者之间取得平衡
- 相比360°全向：方向信息明确，无需复杂推算
- 相比单向扫描：环境覆盖更全，无需频繁旋转

**工作机制：**
```
机器人沿过道行走时：
  → 四向扫描同时工作
  → 左向：扫描左侧书架
  → 右向：扫描右侧书架
  → 前向：预警障碍物
  → 后向：记录已覆盖区域
```

**数据流：**
```
[RFID Scanner] → [融合算法] → [图书位置更新]
    ↓                    ↓              ↓
四个Topic            位置推断       SQLite数据库
```

### 2. 导航与路径规划系统

**挑战：** 图书馆环境特点
- 狭窄过道（1.2m宽）
- 动态障碍物（读者）
- 静态环境（书架）

**解决方案：**
- **代价地图**：多层设计
  - 静态层：书架、墙壁
  - 膨胀层：安全边界
  - 动态层：行人、移动椅子
  - 社会层：优先级决策

- **速度控制**：
  - 过道区：0.3 m/s（安静、谨慎）
  - 开阔区：0.5 m/s
  - 接近目标：0.1 m/s（精确停靠）

- **恢复行为：** 旋转重定位 → 清除代价 → 后退重试

### 3. 任务管理系统

**任务类型：**
- **FindBook**: 寻找特定图书
- **Inventory**: 库存盘点
- **NavigateTo**: 导航到指定地点

**队列管理：**
- 当前：FIFO队列（简单可靠）
- 未来：可扩展为优先级队列

**状态机：**
```
[Pending] → [In Progress] → [Completed / Failed]
    ↓              ↓
   Cancel       Pause / Resume
```

### 4. 数据管理系统

**核心表设计：**

**books表** - 静态信息 + 动态状态
- 图书元数据（ID、标题、作者）
- 位置信息（区域、层、位置）
- 状态（on_shelf / borrowed / misplaced）

**inventory_log表** - 操作历史
- 每次盘点记录
- 预期 vs 实际位置
- 异常标记（错架、缺失）

**task_queue表** - 任务记录
- 待执行、执行中、已完成
- 结果和错误信息

**数据流：**
```
[感知系统] → [位置更新] → [books表]
     ↓              ↓
[库存盘点] → [日志记录] → [inventory_log表]
     ↓              ↓
[任务执行] → [状态更新] → [task_queue表]
```

## 🔄 关键工作流程

### 流程1：图书检索（FindBook）

```
用户请求 → 查询数据库 → 获取图书位置
   ↓
Nav2规划路径 → 机器人移动 → 接近书架（<0.5m）
   ↓
四向RFID扫描 → 检测目标标签 → 确认图书存在
   ↓
屏幕显示位置 → 激光指示 → 等待人工取书
   ↓
更新数据库状态 → 完成任务
```

**成功条件：**
- 导航误差 < 0.5m
- RFID检测成功（或多次尝试后成功）
- 状态更新完成

### 流程2：库存盘点（Inventory）

```
启动任务 → 获取盘点区域 → 生成遍历路径
   ↓
沿过道移动 → 四向扫描持续工作 → 检测所有标签
   ↓
比对数据库 → 标记错架/缺失图书 → 记录日志
   ↓
生成盘点报告 → 更新数据库
```

**关键指标：**
- 扫描覆盖率：100%
- 检测准确率：>85%
- 盘点速度：~2分钟/排书架

## 📊 性能指标

### 导航性能
- **到达精度**：误差 < 0.5m
- **避障成功率**：100%（无碰撞）
- **路径效率**：实际路径长度 / 理论最短路径 < 1.2
- **重定位成功率**：>95%（AMCL参数调优后）

### 感知性能
- **RFID检测率**：>85%（含噪声模型）
- **误检率（False Positive）**：<5%
- **漏检率（False Negative）**：<15%
- **检测延迟**：<0.5秒

### 系统性能
- **启动时间**：<30秒（Gazebo + 所有节点）
- **响应延迟**：任务提交到机器人移动 <2秒
- **数据库查询**：单条查询 <100ms

## 🤝 协作模式

### 人机协作
**机器人职责：**
- 精确导航到目标位置
- 图书精确定位
- 位置指示和信息显示

**人类职责：**
- 从书架取放图书
- 处理异常情况（如图书被遮挡）
- 验证机器人指示

**协作流程：**
```
机器人: 已到达A区3号书架，目标图书在第二层，从左数第5本
        [激光指向具体位置]

人类:   确认图书存在，取走图书

机器人: 检测到图书已取走，更新状态为"borrowed"
```

### 异常处理
**机器人检测不到图书时：**
1. 报告"图书可能不在架"
2. 建议人工验证
3. 更新状态为"possibly_missing"
4. 加入异常日志

## 📌 关键设计权衡

### 权衡1：仿真逼真度 vs 实现复杂度
**选择：** 中等逼真度
- 包含距离-概率RFID模型
- 包含动态障碍物
- 但不过度复杂化视觉识别

**原因：** 在仿真中验证核心逻辑，而非完全复制真实世界

### 权衡2：性能 vs 功能完备
**选择：** 功能优先
- 优先实现完整的工作流程
- 性能劣化问题在Phase 3优化

**原因：** 快速验证概念和功能完整性更重要

### 权衡3：灵活性 vs 约束
**选择：** 参数化设计
- 随机生成图书（支持任意数量）
- 可调书架配置
- 可配置环境参数

**原因：** 便于测试不同规模和场景

### 增：性能与优化决策（基于后续讨论）

#### 图书与书架数量（性能考量后）

| 维度 | 原方案 | 新方案 | 理由 |
|------|--------|--------|------|
| **默认图书数量** | 100本 | **50-100本** | 启动时间~15秒，性能与功能平衡 |
| **最大图书数量** | 1000本 | **500本** | 压力测试上限，避免仿真卡顿 |
| **书架数量** | 60个 | **24个（默认）** | 4排×6个，满足50-100本书放置 |
| **动态障碍物** | 未指定 | **3-4个** | 1移动读者+1静止读者+1-2可移动椅子 |

#### 关键性能优化策略

**优化1：禁用图书碰撞检测**
- **原因：** 图书放置在书架上，不会阻挡机器人路径
- **收益：** 减少90%物理计算量
- **实现：** 图书模型设置为视觉only

**优化2：参数化实例化**
- **图书：** 1个SDF模板 + 100个配置实例
- **书架：** 1个参数化模板，生成24个实例
- **收益：** 内存占用减少60%，加载时间减少50%

**优化3：精确控制障碍物数量**
- 仅3-4个动态障碍物，控制性能影响
- 每个Actor增加CPU负载~2-3%
- 总增加8-12%，在可接受范围

#### 性能目标

| 指标 | 目标值 |
|------|--------|
| 启动时间 | <15秒（默认配置） |
| 运行帧率 | >30 FPS |
| 内存占用 | <2GB |
| 导航响应 | <2秒 |

### 增：BehaviorTree执行框架设计（基于讨论）

#### BT框架选择
- **选择：** 使用独立BehaviorTree.CPP库（非Nav2内置）
- **实现：** 创建`bt_manager_node`，主循环中调用`tree.tick()`
- **频率：** 10Hz（每100ms执行一次）
- **配置：** XML文件（findbook_bt.xml, inventory_bt.xml等）

#### BT与Nav2的关系
```
BT层（顶层决策）：
  └─ bt_manager_node（调用tree.tick()）
      └─ NavigateToBookshelf（Action节点）
          └─ 调用: /navigate_to_pose (Nav2 Action)

Nav2层（运算层）：
  └─ nav2_bt_navigator（Nav2自带的BT）
      └─ 内部也使用BehaviorTree管理导航子任务
      └─ 提供: /navigate_to_pose Action接口
```

**关键点：**
- Nav2拥有自己的BT，用于执行具体导航任务
- 我们的BT在更高层次，是"调用Nav2"的决策
- 两层BT通过ROS2 Action接口解耦

#### BT节点命名规范（暂定）
- **ROS2 Nodes（运算层）：**
  - `rfid_processor_node`
  - `nav2_bt_navigator`
  - `db_manager_node`
  - `ui_manager_node`

- **BT Nodes（决策层）：**
  - `IsBookAvailable`（首字母大写，无空格）
  - `QueryDatabase`
  - `NavigateToBookshelf`
  - `ScanForBook`

#### 错误处理规范
**运算层 → BT层：**
```yaml
运算层错误示例（Database Error）:
- 原始错误: PostgreSQL connection timeout
- 包装后: {service_result: false, error_code: DB_TIMEOUT, error_msg: "数据库连接超时"}

BT层处理:
- 接收包装后错误
- DetermineFailureNode: 根据error_code分类
- 返回: FAILURE（向上传递）
- BT高层: Retry装饰器决定是否重试
```

**BT层只返回三种状态：**
- SUCCESS：任务目标达成
- FAILURE：任务目标无法达成（需重试或终止）
- RUNNING：正在执行中

#### BT配置管理
- **配置文件：** XML格式
- **文件位置：** `config/behaviortrees/findbook_bt.xml`
- **加载方式：** bt_manager_node启动时加载
- **热重载：** Phase 2+可考虑支持（无需重启节点）

#### BT调试支持
- **可视化工具：** Groot（BehaviorTree.CPP官方编辑器）
- **日志级别：**
  - 节点执行：DEBUG
  - 状态转移：INFO
  - 失败/错误：WARN/ERROR
- **ROS2集成：**
  - 发布: /bt_execution_flow（可视化当前执行节点）
  - 发布: /bt_tree_snapshot（当前树状态快照）

#### 架构优势
1. **清晰分层：** 决策与运算分离
2. **测试友好：** 可独立测试BT决策逻辑（用Mock运算层）
3. **可维护：** 修改决策只需改XML，不改动运算代码
4. **可视化：** Groot编辑和实时监控
5. **可扩展：** 添加新决策节点只需继承BT::TreeNode

通信接口汇总

Topics (高频数据流):
  - /rfid/scan/front        (RFIDScan, 10Hz)     - 前向RFID扫描结果
  - /rfid/scan/back         (RFIDScan, 10Hz)     - 后向RFID扫描结果
  - /rfid/scan/left         (RFIDScan, 10Hz)     - 左向RFID扫描结果
  - /rfid/scan/right        (RFIDScan, 10Hz)     - 右向RFID扫描结果
  - /rfid/detection_status  (RFIDTagStatus, 5Hz) - RFID融合检测结果
  - /books/locations        (BookLocationArray, 1Hz) - 所有图书位置
  - /nav2/feedback          (NavigationFeedback, 10Hz) - 导航进度
  - /tf                     (TFMessage, 实时)     - 坐标变换
  - /ui/display_info        (UIDisplayInfo, 5Hz)  - UI显示信息

Services (请求-响应):
  - /db/query_book_info     (QueryBookInfo)      - 查询图书信息
  - /db/update_book_location (UpdateBookLocation) - 更新图书位置
  - /db/insert_log          (InsertLog)          - 插入日志记录
  - /ui/show_marker         (ShowMarker)         - 显示视觉标记
  - /monitor/set_mode       (SetMonitorMode)     - 设置监控模式

Actions (长时间任务):
  - /find_book              (FindBook)           - 找书任务
  - /perform_inventory      (PerformInventory)   - 库存盘点任务
  - /navigate_to_pose       (NavigateToPose)     - 导航任务（Nav2）
